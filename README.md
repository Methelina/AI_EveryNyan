# 🐱 AI_EveryNyan v0.16.1 — Smart Desktop Assistant with Hybrid Memory

> *"Haru, everynyan! How are you!? Fine, thank you! I wish I were a bird!"*

**AI_EveryNyan** — это десктопное AI-приложение с продвинутой архитектурой памяти. В отличие от обычных чат-ботов, которые забывают историю при переполнении контекста, EveryNyan ведет "цифровой дневник", суммирует разговоры и способна возвращаться к старым темам благодаря семантическому поиску.

Приложение построено на Python, использует локальные модели (через Ollama или LLaMA-сервер) и гибридную систему хранения данных: **DuckDB** для точной истории и **Qdrant** для семантических ассоциаций.  
**Текущая версия:** v0.16.1 — с полностью унифицированной поддержкой двух бэкендов, стримингом ответов и исправленным критическим багом в LLaMA-режиме.

---

## ✨ Ключевые особенности

### 🧠 Гибридная архитектура памяти (DuckDB + Qdrant)
Система разделяет хранение на два уровня для максимальной эффективности:
*   **DuckDB (SQL):** Хранит структурированную хронологию чата, дневниковые записи и точные метаданные.
*   **Qdrant (Vector DB):** Хранит эмбеддинги (векторы) диалогов и размышлений. Обеспечивает RAG (Retrieval-Augmented Generation) — поиск информации по смыслу.

### ⏳ Sliding Window & Smart Dump (Скользящее окно)
Уникальный алгоритм управления контекстом:
1.  Когда буфер переполняется, ИИ пишет запись в дневник (через специальный промпт), выделяя суть, эмоции и сущности.
2.  Текст разбивается на смысловые блоки, проверяется на плагиат (через Qdrant) и сохраняется.
3.  Оперативная память очищается, диалог продолжается с чистого листа, но знание сохранено навсегда.

### 📋 Структурированные метаданные дневника (JSON + циркумплексная модель)
Каждая запись дневника теперь содержит не только текст, но и извлечённые LLM метаданные:
- Сущности (`entities`), теги (`topics`), поисковые фразы (`retrieval_cues`).
- **Аффект:** валентность (−1…+1) и возбуждение (−1…+1) по модели Рассела, плюс текстовая метка эмоции.
- Важность, отношения, противоречия и ключевые факты.

### 🛡️ Graceful Shutdown (Безопасное завершение)
Приложение перехватывает сигналы закрытия. Перед выходом оно принудительно вызывает `dump_context_to_memory`, чтобы ни одна мысль не потерялась при сбое.

### 🚫 Анти-повтор (Semantic Anti-Repeat)
Система вычисляет косинусное сходство новых сообщений с историей. Если ИИ начинает повторяться (average similarity > 0.73 или max > 0.69), приложение детектирует петлю и меняет тему.

### 👁️ AI Internal Thoughts UI
Панель "Внутренних мыслей" в DearPyGui показывает процессы в реальном времени: *📝 Summarizing...*, *⚠️ Context overflow!*, *🚫 Skipped duplicate...*, а также отображает `reasoning_content` моделей (если доступно).

### 🔌 Два бэкенда: Ollama и LLaMA (OpenAI-совместимые)
- **Ollama** — основной режим, полная поддержка стриминга и метаданных.
- **LLaMA** — режим для локальных серверов (например, `llama-server`). В v0.16.1 полностью исправлен критический баг, из-за которого ответы не сохранялись и не отображались.

### 🧹 Лемматизация для RAG (через spaCy)
Текст сообщений и дневников перед индексацией обрабатывается через `ru_core_news_sm` и `en_core_web_sm`, что повышает качество поиска по смыслу.

## 🚀 Быстрый старт (Автоматическая установка)

Приложение включает в себя **автоматический инсталлятор** на PowerShell, который выполнит всю черную работу: создаст Conda-окружение, установит зависимости, скачает модель `bge-m3` и сгенерирует конфигурацию.

### Требования
Перед запуском скрипта убедитесь, что на компьютере установлены следующие компоненты:
- **PowerShell** (встроен в Windows)
- **Conda** (Miniforge или Anaconda)
- **Docker Desktop** (для Qdrant)
- **Ollama** (для запуска моделей) или любой LLaMA-сервер с OpenAI‑совместимым API

### 1. Установка
Откройте терминал PowerShell в папке проекта и запустите инсталлятор:

```powershell
.\install_ai_everynyan.ps1
```

**Что делает скрипт:**
*   ✅ Проверяет наличие Git, Conda, Docker и Ollama.
*   ✅ Создает структуру папок (`src`, `data`, `logs`, `config`).
*   ✅ Создает Conda-окружение `env` с Python 3.11.
*   ✅ Устанавливает все Python-зависимости из `requirements.txt`.
*   ✅ Скачивает и устанавливает модель эмбеддингов **`bge-m3`** через Ollama.
*   ✅ Генерирует файл `config/settings.yaml` с правильными настройками (dim=1024).

### 2. Настройка (Опционально)
Инсталлятор создаст файл `config/settings.yaml`. Вы можете выбрать активный бэкенд (`chat_mode: "ollama"` или `"llama"`) и указать свои модели.

```yaml
chat_mode: "ollama"   # или "llama"
ollama:
  chat_model: "deepseek-v3.1:671b-cloud"   # или другая модель
llama:
  base_url: "http://127.0.0.1:8088/v1"
  chat_model: "Falcon-H1R-7B-Q8_0.gguf"
```

> **Важно:** `embedding_model` должен оставаться `bge-m3:latest`, а `embedding_dim` — `1024`, иначе поиск сломается.

### 3. Запуск
Вам потребуются файлы-запускатели (`.bat`) в корне проекта:
1.  **Запустите Qdrant:**
    ```bash
    .\run_qdrant.bat
    ```
2.  **Запустите Ollama** (или ваш LLaMA-сервер):
    ```bash
    ollama serve
    ```
3.  **Запустите приложение:**
    ```bash
    .\run_ai_everynyan.bat
    ```

## 🧩 Как это работает

### 1. Асинхронная архитектура
Интерфейс (DearPyGui) работает в главном потоке, а тяжелые операции (запросы к LLM, сохранение в БД) — в фоновом `asyncio` loop. Окно никогда не зависает, даже если ИИ "думает".

### 2. Пайплайн сообщения (v0.16.1)
1.  **Anti-Repeat:** Проверка, не заходит ли разговор в тупик.
2.  **RAG:** Поиск релевантных воспоминаний в Qdrant (с фильтрацией по порогу схожести).
3.  **Keyword search fallback:** Если векторный поиск не дал результатов, ищем ключевые слова в DuckDB.
4.  **Generation:** Сборка промпта через **LangChain Messages** (System → AI-память → История → User). Это унифицированный способ, работающий одинаково для Ollama и LLaMA.
5.  **Стриминг (LLaMA):** Ответ отображается по мере генерации.
6.  **Save:** Сохранение диалога в DuckDB и Qdrant с извлечением метаданных (сущности, теги, факты).
7.  **Overflow Dump:** Если контекст переполнен -> вызывается суммаризация и сохранение в "Дневник" с JSON-метаданными.

### 3. Дневник и метаданные
При вызове `dump_context_to_memory` используется промпт из `settings.yaml` (`diary.summary_prompt`), который возвращает текст и JSON-блок. `DiaryEntryMetadata` (Pydantic) валидирует поля, включая `affect_valence`, `affect_arousal`, `emotion_label`. Данные сохраняются в DuckDB и Qdrant.

## 🗺️ Дорожная карта (Roadmap)

### Уже реализовано в v0.16.x
- ✅ Полноценная поддержка двух бэкендов (Ollama + LLaMA) через единый LangChain-интерфейс.
- ✅ Стриминг ответов для LLaMA-режима.
- ✅ Структурированные метаданные дневника (JSON + циркумплексная модель аффекта).
- ✅ Лемматизация текста для улучшенного RAG (spaCy).
- ✅ Исправлен критический баг с `None`-ответами в LLaMA-режиме.

### В активной разработке (WIP)
- 🚧 **MCP (Model Context Protocol) + полноценные инструменты**  
  Унифицированный интерфейс для вызова внешних функций: чтение/запись файлов, выполнение shell-команд, интеграция с API. Позволит EveryNyan не просто отвечать, но и действовать.
- 🚧 **SearXNG Web Search**  
  Анонимный метапоиск через собственный экземпляр SearXNG. LLM сможет получать актуальную информацию из интернета без API-ключей и слежки.
- 🚧 **Внутренние размышления (проактивность)**  
  Периодическая генерация случайных мыслей и сохранение их в дневник. ИИ будет «думать» даже без команды пользователя, что повышает живость и связность памяти.

### Планы на ближайшее будущее
- **Confidence & Usage Count**  
  Добавление в метаданные записей полей уверенности (`-1..1`) и счётчика использований. Взвешивание результатов RAG с учётом confidence.
- **Фоновая чистка памяти**  
  Автоматическое удаление ложных (`confidence < -0.5`) и давно неиспользуемых записей (старее 90 дней, `usage_count = 0`).
- **Динамическая адаптация RAG**  
  Автоматическая подстройка `top_k` и `similarity_threshold` на основе обратной связи от LLM (например, частые жалобы на отсутствие информации).
- **Semantic Chunker** — умное разбиение длинных текстов на смысловые блоки вместо обрезания.
- **NER Enrichment** — автоматическое извлечение имён, дат и организаций из дневников (через spaCy или трансформеры).

### Дальняя перспектива (v1.0+)
- **Галерея** — поддержка изображений в сообщениях (drag‑and‑drop, описание через vision‑LLM).
- **Генерация изображений** — интеграция с локальной Stable Diffusion (Automatic1111 или `diffusers`).
- **Напоминания и планировщик** — ИИ сможет устанавливать таймеры и напоминать о событиях.
