# 🐱 AI_EveryNyan v0.4.3 — Smart Desktop Assistant with Hybrid Memory

> *"Haru, everynyan! How are you!? Fine, thank you! I wish I were a bird!"*

**AI_EveryNyan** — это десктопное AI-приложение с продвинутой архитектурой памяти. В отличие от обычных чат-ботов, которые забывают историю при переполнении контекста, EveryNyan ведет "цифровой дневник", суммирует разговоры и способна возвращаться к старым темам благодаря семантическому поиску.

Приложение построено на Python, использует локальные модели (через Ollama) и гибридную систему хранения данных: **DuckDB** для точной истории и **Qdrant** для семантических ассоциаций.

---

## ✨ Ключевые особенности

### 🧠 Гибридная архитектура памяти (DuckDB + Qdrant)
Система разделяет хранение на два уровня для максимальной эффективности:
*   **DuckDB (SQL):** Хранит структурированную хронологию чата, дневниковые записи и точные метаданные.
*   **Qdrant (Vector DB):** Хранит эмбеддинги (векторы) диалогов и размышлений. Обеспечивает RAG (Retrieval-Augmented Generation) — поиск информации по смыслу.

### ⏳ Sliding Window & Smart Dump (Скользящее окно)
Уникальный алгоритм управления контекстом:
1.  Когда буфер переполняется, ИИ пишет запись в дневник (через специальный промпт), выделяя суть, эмоции и сущности.
2.  Текст разбивается на смысловые блоки, проверяется на плагиат (через Qdrant) и сохраняется.
3.  Оперативная память очищается, диалог продолжается с чистого листа, но знание сохранено навсегда.

### 🛡️ Graceful Shutdown (Безопасное завершение)
Приложение перехватывает сигналы закрытия. Перед выходом оно принудительно вызывает `dump_context_to_memory`, чтобы ни одна мысль не потерялась при сбое.

### 🚫 Анти-повтор (Semantic Anti-Repeat)
Система вычисляет косинусное сходство новых сообщений с историей. Если ИИ начинает повторяться (similarity > 0.73), приложение детектирует петлю и меняет тему.

### 👁️ AI Internal Thoughts UI
Панель "Внутренних мыслей" в DearPyGui показывает процессы в реальном времени: *📝 Summarizing...*, *⚠️ Context overflow!*, *🚫 Skipped duplicate...*

## 🚀 Быстрый старт (Автоматическая установка)

Приложение включает в себя **автоматический инсталлятор** на PowerShell, который выполнит всю черную работу: создаст Conda-окружение, установит зависимости, скачает модель `bge-m3` и сгенерирует конфигурацию.

### Требования
Перед запуском скрипта убедитесь, что на компьютере установлены следующие компоненты:
- **PowerShell** (встроен в Windows)
- **Conda** (Miniforge или Anaconda)
- **Docker Desktop** (для Qdrant)
- **Ollama** (для запуска моделей)

### 1. Установка
Откройте терминал PowerShell в папке проекта и запустите инсталлятор:

```powershell
.\install_ai_everynyan.ps1
```

**Что делает скрипт:**
*   ✅ Проверяет наличие Git, Conda, Docker и Ollama.
*   ✅ Создает структуру папок (`src`, `data`, `logs`, `config`).
*   ✅ Создает Conda-окружение `env` с Python 3.11.
*   ✅ Устанавливает все Python-зависимости из `requirements.txt`.
*   ✅ Скачивает и устанавливает модель эмбеддингов **`bge-m3`** через Ollama.
*   ✅ Генерирует файл `config/settings.yaml` с правильными настройками (dim=1024).

### 2. Настройка (Опционально)
Инсталлятор создаст файл `config/settings.yaml`. Если вы хотите использовать другую LLM-модель для общения (вместо стандартной `qwen2.5:7b`), отредактируйте этот файл:

```yaml
llm:
  chat_model: "имя_вашей_модели"
```

> **Важно:** Не меняйте `embedding_model: bge-m3` и `embedding_dim: 1024`, если вы не уверены в том, что делаете. Инсталлятор уже настроил систему под эту конкретную модель.

### 3. Запуск
Вам потребуются файлы-запускатели (`.bat`) в корне проекта:
1.  **Запустите Qdrant:**
    ```bash
    .\run_qdrant.bat
    ```
2.  **Запустите приложение:**
    ```bash
    .\run_ai_everynyan.bat
    ```
    *(Убедитесь, что Ollama работает: `ollama serve`)*

## 🧩 Как это работает

### 1. Асинхронная архитектура
Интерфейс (DearPyGui) работает в главном потоке, а тяжелые операции (запросы к LLM, сохранение в БД) — в фоновом `asyncio` loop. Окно никогда не зависает, даже если ИИ "думает".

### 2. Пайплайн сообщения
1.  **Anti-Repeat:** Проверка, не заходит ли разговор в тупик.
2.  **RAG:** Поиск релевантных воспоминаний в Qdrant.
3.  **Generation:** Сборка промпта (Система + Память + История) и запрос к LLM.
4.  **Save:** Сохранение в DuckDB и Qdrant.
5.  **Overflow Dump:** Если контекст переполнен -> вызывается суммаризация и сохранение в "Дневник".
