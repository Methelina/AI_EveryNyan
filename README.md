# 🐱 AI_EveryNyan — Интеллектуальный чат-ассистент с семантической памятью

> *"I wish I were a bird — free to remember, reflect, and respond."*

**AI_EveryNyan** — это десктопный AI-агент с архитектурой, вдохновлённой передовыми практиками в области агентных систем. Приложение сочетает интерактивный чат, долговременную семантическую память (RAG) и модульную систему инструментов для выполнения сложных задач.

Проект находится в активной разработке. Текущая версия предоставляет рабочий базовый функционал, а архитектура заложена с расчётом на постепенное внедрение расширенных возможностей.

---

## ✨ Текущий функционал (работает сейчас)

- **💬 Чат с поддержкой контекста**  
  Интерфейс на DearPyGui с историей сообщений, тёмной темой и асинхронной обработкой запросов.

- **🧠 Семантическая память на Qdrant**  
  Все диалоги автоматически векторизуются и сохраняются в локальную векторную БД. При похожих запросах система извлекает релевантные фрагменты прошлых бесед.

- **🔍 RAG (Retrieval-Augmented Generation)**  
  Перед формированием ответа AI выполняет семантический поиск в памяти и обогащает промпт найденным контекстом.

- **⚙️ Гибкая конфигурация**  
  Настройки через YAML-файл: выбор LLM-бэкенда (Ollama / OpenAI-compatible), моделей для чата и эмбеддингов, параметров GUI и логирования.

- **🐳 Простая установка зависимостей**  
  Qdrant запускается в Docker-контейнере одной командой. Остальное устанавливается через Conda.

## 🚀 Быстрый старт

### Требования
- **Conda** (Miniforge / Anaconda)
- **Docker Desktop** (для Qdrant)
- **Ollama** или любой OpenAI-compatible LLM-сервер
- **Git**, **PowerShell** (Windows)

### Установка
```powershell
# В папке проекта:
.\install_ai_everynyan.ps1
```

### Настройка
1. Отредактируйте `config/settings.yaml`:
   ```yaml
   llm:
     base_url: "http://localhost:11434/v1"  # Ollama
     chat_model: "qwen2.5:7b"
     embedding_model: "mxbai-embed-large:latest"
   vector_db:
     embedding_dim: 1024  # Важно: должно соответствовать модели эмбеддингов!
   ```

2. Загрузите модели в Ollama:
   ```bash
   ollama pull qwen2.5:7b
   ollama pull mxbai-embed-large:latest
   ```

### Запуск
```powershell
# 1. Запустите Qdrant (в отдельном окне):
.\run_qdrant.bat

# 2. Убедитесь, что Ollama запущен:
ollama serve

# 3. Запустите чат:
.\run_ai_everynyan.bat
```

## 🎯 Целевая архитектура и дорожная карта

Проект развивается в направлении полноценного агентного фреймворка. Ниже перечислены ключевые компоненты, реализация которых запланирована:

### 🔧 Модульная система инструментов (Tool Framework)
- **Динамическая регистрация инструментов** с валидацией параметров через JSON Schema
- **Базовые инструменты:**
  - `ask_diary` — семантический запрос к памяти с обогащением контекста
  - `ask_web` — веб-поиск через внешние API (Tavily, SerpAPI)
  - `pause` / `wait` — управление потоком диалога
- **Расширяемость**: возможность добавления кастомных инструментов без изменения ядра

### 🧠 Продвинутая система памяти
- **Адаптивный порог релевантности**: динамическая настройка чувствительности поиска в зависимости от контекста
- **Анти-плагиат**: проверка новых записей на семантическое дублирование перед сохранением
- **"Спящая консолидация"** (Sleeping Consolidation): фоновый процесс периодической рефлексии над памятью — объединение, уточнение и структурирование записей через LLM
- **Ленивая генерация эмбеддингов**: вычисление векторов только при первом запросе, с кэшированием в БД

### 🎨 Генерация изображений (планируется)
- **Итеративный пайплайн**:
  1. Инженерия промпта через LLM с учётом стиля персонажа
  2. Генерация через Stable Diffusion (локально или по API)
  3. Оценка качества через Vision-модель
  4. Обратная связь → уточнение промпта → повтор (до 10 итераций)
- **Сохранение дизайна персонажа**: механизмы предотвращения дрейфа внешности при многократной генерации
- **Галерея**: локальное хранилище с семантическим поиском по описаниям

### ⚡ Агентная оркестрация
- **Управление контекстом**: автоматическое сжатие истории, сброс в память при достижении лимита токенов
- **Проактивное поведение**: случайные "пробуждения" для инициации диалога или рефлексии
- **Обработка ошибок**: graceful fallback, повторные попытки, логирование сценариев сбоев
- **Асинхронная архитектура**: неблокирующая обработка всех операций (GUI, LLM, БД)

### 🔐 Конфигурация и безопасность
- **Разделение конфигов**: шаблон (`settings.yaml.example`) в репозитории, личные настройки — локально
- **Поддержка переменных окружения** для чувствительных данных (API-ключи)
- **Изоляция данных**: все пользовательские файлы хранятся в `data/`, не затрагивая системные пути
