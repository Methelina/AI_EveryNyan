# 🐱 AI_EveryNyan v0.16.2 — Smart Desktop Assistant with Hybrid Memory & MCP Tools

> *"Haru, everynyan! How are you!? Fine, thank you! I wish I were a bird!"*

**AI_EveryNyan** — это десктопное AI-приложение с продвинутой архитектурой памяти и поддержкой внешних инструментов через MCP (Model Context Protocol). В отличие от обычных чат-ботов, EveryNyan ведет "цифровой дневник", суммирует разговоры, может искать информацию в интернете (через SearXNG) и извлекать содержимое веб-страниц.

Приложение построено на Python, использует локальные модели (Ollama или LLaMA-сервер) и гибридную систему хранения: **DuckDB** для точной истории и **Qdrant** для семантических ассоциаций.  
**Текущая версия:** v0.16.2 — с полностью унифицированной поддержкой двух бэкендов, стримингом ответов, MCP агентом, цветным логированием вызовов инструментов и исправленным критическим багом в LLaMA-режиме.

## ✨ Ключевые особенности

### 🧠 Гибридная архитектура памяти (DuckDB + Qdrant)
Система разделяет хранение на два уровня:
*   **DuckDB (SQL):** Структурированная хронология чата, дневниковые записи и точные метаданные.
*   **Qdrant (Vector DB):** Эмбеддинги диалогов и размышлений. Обеспечивает RAG (Retrieval-Augmented Generation) — поиск информации по смыслу.

### ⏳ Sliding Window & Smart Dump
Уникальный алгоритм управления контекстом:
1.  Когда буфер переполняется, ИИ пишет запись в дневник, выделяя суть, эмоции и сущности.
2.  Текст разбивается на смысловые блоки, проверяется на плагиат (через Qdrant) и сохраняется.
3.  Оперативная память очищается, диалог продолжается с чистого листа, но знание сохранено навсегда.

### 📋 Структурированные метаданные дневника (JSON + циркумплексная модель)
Каждая запись дневника содержит извлечённые LLM метаданные:
- Сущности (`entities`), теги (`topics`), поисковые фразы (`retrieval_cues`).
- **Аффект:** валентность (−1…+1) и возбуждение (−1…+1) по модели Рассела, плюс текстовая метка эмоции.
- Важность, отношения, противоречия и ключевые факты.

### 🛡️ Graceful Shutdown
Приложение перехватывает сигналы закрытия. Перед выходом оно принудительно вызывает `dump_context_to_memory`, чтобы ни одна мысль не потерялась.

### 🚫 Анти-повтор (Semantic Anti-Repeat)
Косинусное сходство новых сообщений с историей. Если ИИ начинает повторяться (average similarity > 0.73 или max > 0.69), детектируется петля и тема меняется.

### 👁️ AI Internal Thoughts UI
Панель "Внутренних мыслей" в DearPyGui показывает процессы в реальном времени: RAG-запросы, вызовы инструментов, ошибки, reasoning_content моделей. В v0.16.2 добавлено **цветное логирование MCP инструментов**: вызовы (жёлтый) и результаты (зелёный/красный).

### 🔌 Два бэкенда: Ollama и LLaMA
- **Ollama** — основной режим, полная поддержка стриминга и метаданных.
- **LLaMA** — режим для локальных серверов (например, `llama-server`). В v0.16.1 полностью исправлен критический баг сохранения ответов, в v0.16.2 добавлен fallback на прямой LLM при ошибках агента.

### 🧹 Лемматизация для RAG (через spaCy)
Текст сообщений и дневников перед индексацией обрабатывается через `ru_core_news_sm` и `en_core_web_sm`, что повышает качество поиска.

### 🔧 MCP + Веб-инструменты (v0.16.2)
Интегрированы инструменты через **Model Context Protocol**:
- **Веб-поиск через SearXNG** — анонимный метапоиск (без API-ключей).
- **Извлечение содержимого URL** — с поддержкой трёх режимов: `legacy` (httpx+bs4), `playwright` (полный браузер), `nodriver` (лёгкий CDP). Автоматическое очищение HTML → Markdown.
- **Цветное логирование** всех вызовов инструментов и результатов в панели SYSTEM LOG.
- **Fallback при ошибках агента** — если MCP агент упал, система переключается на прямой вызов LLM.

## 🚀 Быстрый старт (Автоматическая установка)

Приложение включает автоматический инсталлятор на PowerShell.

### Требования
- **PowerShell**
- **Conda** (Miniforge или Anaconda)
- **Docker Desktop** (для Qdrant)
- **Ollama** или любой LLaMA-сервер с OpenAI‑совместимым API

### 1. Установка
```powershell
.\install_ai_everynyan.ps1
```

**Что делает скрипт:**
- Проверяет Git, Conda, Docker, Ollama.
- Создаёт структуру папок и Conda-окружение `env` с Python 3.11.
- Устанавливает зависимости из `requirements.txt`.
- Скачивает модель эмбеддингов **`bge-m3`** через Ollama.
- Генерирует `config/settings.yaml` (dim=1024).

### 2. Настройка (Опционально)
Отредактируйте `config/settings.yaml`. Пример:
```yaml
chat_mode: "ollama"   # или "llama"
ollama:
  chat_model: "qwen2.5:7b"
llama:
  base_url: "http://127.0.0.1:8088/v1"
  chat_model: "Falcon-H1R-7B-Q8_0.gguf"
# Для MCP инструментов (SearXNG)
searxng_url: "http://localhost:2597"
```

> **Важно:** `embedding_model` должен быть `bge-m3:latest`, `embedding_dim` — `1024`.

### 3. Запуск
1. **Qdrant:** `.\run_qdrant.bat`
2. **Ollama / LLaMA-сервер**
3. **Приложение:** `.\run_ai_everynyan.bat`

## 🧩 Как это работает

### 1. Асинхронная архитектура
GUI (DearPyGui) в главном потоке, LLM-запросы и RAG — в фоновом `asyncio` loop.

### 2. Пайплайн сообщения (v0.16.2)
1. **Anti-Repeat** — проверка на семантические петли.
2. **RAG** — поиск в Qdrant (с порогом схожести).
3. **Keyword search fallback** — если векторов нет, ищем в DuckDB.
4. **MCP Agent** — агент LangGraph с инструментами. Вызовы и результаты логируются цветом в панели мыслей.
5. **Fallback** — при ошибке агента → прямой LLM (Ollama или LLaMA).
6. **Стриминг** (LLaMA) — ответ появляется по токенам.
7. **Сохранение** диалога в DuckDB и Qdrant с извлечением метаданных.
8. **Smart Dump** — при переполнении контекста пишется дневник с JSON-метаданными.

### 3. MCP инструменты
- `web_search(query, categories, language, max_results)` — поиск через SearXNG, результат в Markdown.
- `fetch_url(url, max_length)` — скачивание страницы, очистка, конвертация в Markdown. Режим выбирается переменной `FETCH_MODE` (legacy/playwright/nodriver).
